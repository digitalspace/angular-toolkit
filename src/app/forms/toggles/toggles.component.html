Toggle Inputs

NGDS toggles are comprised of checkboxes and switches and are good for representing boolean controls. Note that NGDS toggles require an underlying Angular FormControl object and therefore may be overkill for some toggle applications. Rudumentary toggles can be implemented with raw bootstrap.

Basic checkbox

NGDS toggle inputs can be true (checked) or false (unchecked).

Basic switch

The default toggle type is a checkbox.

Pass in [switch]="true" to change the toggle to a switch.

Toggle with standard label

By default, NGDS toggle labels are positioned to the right of the toggle instead of above standard like other controls. If the above standard is desired, manually adding a ngds-input-header is possible.

Disabled toggle

Disabled and loading state can be controlled with [disabled] and [loadWhile] attributes, respectively. You can also disable the control programmatically by calling control.disable().

Toggle validation

Angular Validators will work with NGDS toggles. requireTrue is common for toggle inputs.

Inline toggle

NGDS's ngdsInputPrepend and ngdsInputAppend can be leveraged to show checkboxes inline. Inline text looks similar to a toggle [label] but is unclickable by default.

Clickable prepends and appends

The clickable behaviour of ngdsInputPrepend and ngdsInputAppend can be altered using [clickablePrepend]="true" and [clickableAppend]="true".

When set to true, the input will toggle when the prepended/appended elements are clicked.
